    2. Async I/O â€“ Keep run() async to avoid blocking orchestrator threads when nodes are co-located.
    3. Environment contracts â€“ Node images expose:
    4. Health endpoint at /healthz (HTTP 200 = OK)
    5. Metadata at /openapi.json so orchestrator can discover inputs/outputs.
    6. Versioning â€“ Bump image tags with semantic versioning; orchestrator chooses compat image based on chain requirements.
